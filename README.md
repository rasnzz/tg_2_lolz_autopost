# Автопостер из Telegram в Lolz.live

Система, которая автоматически собирает посты из публичных Telegram-каналов и публикует их на форумах lolz.live.

## Особенности

- Парсит публичные Telegram-каналы без использования Telegram API
- Фильтрует посты по ключевым словам, минимальной/максимальной длине
- Сохраняет оригинальное форматирование текста (пробелы, переносы строк, табуляции)
- Извлекает и вставляет изображения в виде BBCode [img] ссылок
- Обеспечивает задержки между постами (минимум 300 секунд, настраиваемо)
- Проверяет дубликаты с помощью ID поста или хеша содержимого
- Корректно обрабатывает ошибки API и ограничения по частоте запросов

## Требования

- Python 3.7+
- Зависимости перечислены в requirements.txt

## Установка

1. Клонируйте этот репозиторий
2. bash - python3 -m venv venv
3. bash - source venv/bin/activate
4. Установите зависимости:
   bash - pip install -r requirements.txt

## Использование

Запустите главный скрипт и следуйте интерактивным подсказкам:

```bash
python main.py
```

Вам будут заданы вопросы:
- токен API lolz.live (получите его на https://lolz.live/account/api)
- Telegram-каналы (через запятую, например, @channel1, @channel2)
- ID целевых форумов (через запятую, например, 128, 876)
- Стоп-слова (опционально, через запятую)
- Минимальная длина поста (по умолчанию 50)
- Максимальная длина поста (по умолчанию 2000)
- Задержка между постами в секундах (по умолчанию 300)
- Префикс заголовка (опционально)

## Конфигурация

При каждом запуске система:
1. Удаляет старый файл config.yaml
2. Запрашивает все настройки интерактивно
3. Создает новый файл config.yaml
4. Обрабатывает посты в соответствии с конфигурацией

## Безопасность

- Токены API хранятся только в config.yaml
- Файл конфигурации не коммитится в систему контроля версий
- Токены не записываются в логи

## Обработка ошибок

- Ошибки ограничения частоты запросов (429) обрабатываются с автоматическими повторами
- Ошибки сервера (5xx) обрабатываются с экспоненциальным затуханием
- Ошибки клиента (400, 401, 403, 404) записываются в лог и пропускаются
- Ошибки сети обрабатываются корректно

## Ограничения

- Система обеспечивает минимальную задержку в 300 секунд между постами
- Ограничения по частоте запросов к API соблюдаются (300 запросов в минуту)
- Содержимое поста ограничено 5000 символами
